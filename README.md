# 🎙️ سیستم پردازش صوت Real-time

[![Python](https://img.shields.io/badge/Python-3.10-blue)](https://www.python.org/)
[![Gradio](https://img.shields.io/badge/Gradio-UI-orange)](https://gradio.app/)
[![License](https://img.shields.io/badge/License-MIT-green)](https://opensource.org/licenses/MIT)

یک جعبه‌ابزار همه‌جانبه و پیشرفته برای پردازش، افکت‌گذاری، جداسازی و تبدیل صدای هوشمند (Voice Conversion) با استفاده از مدل‌های هوش مصنوعی پیشرفته RVC v2 و کتابخانه‌های پردازش سیگنال دیجیتال (DSP) به همراه رابط کاربری مدرن تحت وب.

---

## 📝 معرفی پروژه

پروژه **سیستم پردازش صوت Real-time** ابزاری کاربردی و قدرتمند برای تولیدکنندگان محتوا، مهندسان صدا، توسعه‌دهندگان حوزه هوش مصنوعی صوتی و علاقه‌مندان به ویرایش و شبیه‌سازی صدا است. این سیستم به کاربران اجازه می‌دهد که فایل‌های صوتی خود را آپلود کرده یا صدای خود را به صورت مستقیم از طریق میکروفون ضبط کنند، و سپس انواع پردازش‌های صوتی را با استفاده از رابط گرافیکی وب اعمال نمایند.

این پروژه تلفیقی از متدهای سنتی پردازش سیگنال (DSP) و تکنولوژی مدرن شبیه‌سازی صدا با یادگیری عمیق (Deep Learning) است که با بهینه‌سازی‌های انجام شده، قابلیت اجرا روی سیستم‌های فاقد کارت گرافیک مجزا (اجرای روان روی CPU) را نیز دارا می‌باشد.

## ✨ قابلیت‌ها (Features)

| قابلیت                             | آیکون | توضیحات                                                                                                  |
| :--------------------------------- | :---: | :------------------------------------------------------------------------------------------------------- |
| **تغییر صدا (Voice Conversion)**   |  🔊   | شبیه‌سازی و تبدیل صدای ورودی به صدای خواننده‌ها یا شخصیت‌های معروف با استفاده از مدل‌های RVC v2 (نسخه ۲) |
| **تغییر زیر و بمی (Pitch Shift)**  |  🎵   | تغییر تن و گام صدا به صورت نیم‌پله (Semitone) بدون تغییر در سرعت و طول صدا                               |
| **اکولایزر (EQ Filter)**           |  🎛️   | تقویت یا تضعیف فرکانس‌های خاص صدا در ۳ باند فرکانسی اصلی: Bass، Mid و Treble                             |
| **کاهش نویز (Noise Reduction)**    |  🔇   | فیلتر کردن نویزهای ثابت پس‌زمینه (مانند صدای فن یا هیس) و نویزهای غیرثابت به کمک کتابخانه `noisereduce`  |
| **تغییر سرعت (Time Stretch)**      |  ⏩   | افزایش یا کاهش سرعت پخش صدا بدون تغییر در گام و فرکانس آن                                                |
| **خط لوله پردازش (Pipeline)**      |  ⚙️   | ترکیب و اعمال همزمان چندین افکت پردازشی بر روی صدا به صورت زنجیره‌ای و متوالی                            |
| **جداسازی وکال (Vocal Isolation)** |  🎤   | تفکیک و جداسازی صدای خواننده (Vocals) از موسیقی پس‌زمینه (Instrumental) با الگوریتم‌های هوشمند Demucs    |
| **دریافت خروجی‌ها**                |  📥   | قابلیت دانلود نتایج نهایی پردازش در دو فرمت استاندارد WAV و MP3                                          |
| **تجسم‌سازی فرکانسی**              |  📊   | نمایش و مقایسه نمودار شکل موج صوتی (Waveform) و طیف فرکانسی (Spectrum) قبل و بعد از پردازش               |

## 🚀 نصب و راه‌اندازی (گام به گام)

### گام اول: شبیه‌سازی مخزن (Clone)

ابتدا پروژه را در سیستم خود کلون کنید و وارد پوشه پروژه شوید:

```bash
git clone https://github.com/Omid-H/audio_project.git
cd audio_project
```

### گام دوم: ساخت محیط مجازی کندا (Conda Environment)

یک محیط مجازی جدید با پایتون نسخه 3.10 بسازید و آن را فعال کنید:

```bash
conda create -n rvc_env python=3.10 -y
conda activate rvc_env
```

### گام سوم: نصب FFmpeg

برای اجرای بدون دردسر ابزار جداسازی وکال و سایر پردازش‌های فرمت فایل صوتی، ابزار FFmpeg را از طریق کندا نصب کنید:

```bash
conda install -c conda-forge ffmpeg -y
```

### گام چهارم: نصب کتابخانه‌ها و پکیج‌های پایتون

وابستگی‌های پایتونی پروژه را با دستورات زیر نصب کنید (برخی پکیج‌ها نیاز است بدون وابستگی‌های تداخلی نصب شوند):

```bash
pip install -r requirements.txt
pip install omegaconf==2.1.1
pip install hydra-core==1.0.7 --no-deps
pip install fairseq==0.12.2 --no-deps
pip install rvc-python --no-deps
pip install setuptools==69.5.1 --force-reinstall
```

### گام پنجم: دانلود مدل‌های صوتی پایه

برای استفاده از بخش تغییر صدا (Voice Conversion)، باید مدل‌های سنگین هوش مصنوعی را با اجرای اسکریپت زیر دانلود کنید:

```bash
python setup_models.py
```

_نکته: در صورتی که فایل‌ها از قبل در پوشه `rvc_models/` وجود داشته باشند، دانلود آنها به صورت خودکار نادیده گرفته می‌شود._

### گام ششم: اجرای برنامه

پس از پایان دانلود مدل‌ها، وب‌اپلیکیشن پروژه را راه‌اندازی کنید:

```bash
python main_app.py
```

پس از مشاهده پیغام موفقیت در ترمینال، مرورگر خود را باز کرده و به آدرس زیر بروید:

## 💡 راهنمای استفاده از بخش‌های مختلف برنامه

### 🔊 Voice Conversion

- **کاربرد:** تغییر صدای شما یا فایل صوتی دلخواه به صدای یکی از شخصیت‌های معروف.
- **روش استفاده:** صدای خود را ضبط کرده یا یک فایل آپلود کنید، مدل صدای مورد نظر را از منوی بازشو انتخاب کنید و روی دکمه **Convert** کلیک کنید.
- **خروجی:** فایل صوتی جدید با همان لحن و بیان اما با طنین صدای شخصیت انتخابی.

### 🎵 Pitch Shift

- **کاربرد:** کلفت‌تر یا نازک‌تر کردن صدا بدون تغییر سرعت.
- **روش استفاده:** فایل ورودی را مشخص کرده و اسلایدر semitone را تغییر دهید (مقادیر منفی صدا را بم‌تر و مردانه‌تر، مقادیر مثبت صدا را زیرتر و زنانه‌تر می‌کنند).
- **خروجی:** صدای تغییر گام داده شده به همراه نمودار فرکانسی جدید.

### 🎛️ EQ Filter

- **کاربرد:** افزایش یا کاهش بخش خاصی از طیف فرکانسی صدا.
- **روش استفاده:** باند فرکانسی مورد نظر (Bass برای فرکانس پایین، Mid برای صدای میانی و Treble برای صدای زیر) را انتخاب کرده و مقدار تضعیف یا تقویت را به دسی‌بل تنظیم کنید.
- **خروجی:** صدای فیلتر شده (به عنوان مثال، بیس بسیار قوی یا صدای تلفنی مانند).

### 🔇 Noise Reduction

- **کاربرد:** پاک‌سازی صدا از نویز پس‌زمینه.
- **روش استفاده:** میزان درصد کاهش نویز (Noise Reduction Ratio) را مشخص کرده و دکمه اعمال را بزنید.
- **خروجی:** خروجی صوتی تمیز و فاقد صدای فن یا نویز استاتیک محیط.

### ⏩ Time Stretch

- **کاربرد:** تغییر سرعت پخش فایل صوتی.
- **روش استفاده:** ضریب سرعت پخش را مشخص کنید (کمی مقادیر زیر ۱.۰ سرعت را کند و مقادیر بالای ۱.۰ سرعت را تند می‌کند).
- **خروجی:** صدایی با سرعت متفاوت بدون کوچک‌ترین تغییر در گام یا فرکانس اصلی.

### ⚙️ Pipeline

- **کاربرد:** اعمال همزمان چند فیلتر به صورت متوالی.
- **روش استفاده:** چک‌باکس‌های افکت‌های مدنظر را فعال کرده و مقادیر آن‌ها را تنظیم کنید، سپس دکمه پردازش را بفشارید.
- **خروجی:** صوتی که تمام مراحل فیلترگذاری (مثلاً ابتدا نویزگیری، سپس تغییر گام و در نهایت افزایش سرعت) را به صورت متوالی طی کرده است.

### 🎤 Vocal Isolation

- **کاربرد:** استخراج صدای خواننده از آهنگ.
- **روش استفاده:** یک فایل موسیقی کامل آپلود کرده و بر روی دکمه جداسازی کلیک کنید.
- **خروجی:** دو فایل مجزا؛ یکی حاوی وکال (صدای خواننده) و دیگری موزیک پس‌زمینه (Instrumental).

## 📂 ساختار پوشه‌های پروژه

نمای درختی از دایرکتوری‌های اصلی پروژه جهت درک بهتر ارتباطات فایل‌ها:

```text
audio_project/
│
├── offline/             # توابع پردازش صوت آفلاین (DSP)
│   ├── file_io.py          # خواندن و نوشتن فایل‌های صوتی
│   ├── pitch_shift.py      # تغییر تن صدا
│   ├── eq_filter.py        # فیلتر اکولایزر باندها
│   ├── noise_reduction.py  # حذف نویزهای ثابت و متغیر
│   ├── time_stretch.py     # تغییر سرعت پخش
│   ├── vocal_isolation.py  # جداسازی خواننده از موزیک (Demucs)
│   └── pipeline.py         # اجرای متوالی افکت‌ها
│
├── rvc_models/          # دایرکتوری نگهداری مدل‌های RVC v2
│   ├── hubert_base.pt      # مدل پایه صوتی اول
│   ├── rmvpe.pt            # مدل تشخیص Pitch دوم
│   └── [Model_Name]/       # پوشه مدل صوتی هر کاراکتر (مانند Arnold)
│
├── samples/             # فایل‌های صوتی نمونه برای تست و عیب‌یابی
├── output/              # پوشه نگه‌داری نتایج خروجی برنامه
├── main_app.py          # برنامه اصلی و رابط وب Gradio
└── setup_models.py      # اسکریپت دانلود خودکار مدل‌ها
```

## ⚠️ نکات مهم

- **مدل‌ها در گیت آپلود نمی‌شوند:** به دلیل محدودیت حجم گیت هاب، مدل‌های حجیم شبیه‌سازی در مخزن آپلود نشده‌اند. حتماً در اولین نصب اسکریپت `setup_models.py` را اجرا کنید.
- **تطابق با CPU:** مدل استنتاج شبیه‌سازی صدا به صورت پیش‌فرض بر روی اجرای با پردازنده مرکزی (CPU) بهینه‌سازی شده است تا همه کاربران فاقد کارت گرافیک مجزای انویدیا (NVIDIA GPU) نیز قادر به اجرای پروژه باشند.
- **دانلود ابزار Demucs در اولین اجرا:** اولین باری که از ویژگی جداسازی وکال استفاده می‌کنید، ابزار Demucs به صورت خودکار مدل اختصاصی خود با حجم حدود ۳۰۰ مگابایت را دانلود خواهد کرد. این فرآیند فقط یک‌بار رخ می‌دهد.
- **مدل‌های صوتی زبان فارسی:** برای بهترین بازدهی در تبدیل کلام فارسی، توصیه می‌شود از مدل‌هایی استفاده کنید که با دیتاست‌های زبان فارسی آموزش دیده‌اند (مانند هایده).
